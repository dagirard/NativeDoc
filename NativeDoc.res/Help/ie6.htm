Google Chrome issue


Google Chrome does not handle very well website that are not placed on a webserver.
Google has decided that website that are not run within a webserver are not allowed to do some calls.

One problem is with the iFrame template, they are calls that attempt to access to the parent frame, and those calls are forbidden by Google in local.
Another problem is with the template with iFrame, Google chrome does not allow to store cookies in local, so the separator size is not stored.

These are very annoying issues and there is a lot of complaint about thoses but Google does not seem to care about that, so:
Do not use Google Chrome (in local) in order to have the best experience with NativeDoc.

It is sad to see this with chrome, as it is a very fast and well written browser.


Internet Explorer 6 support


Internet Explorer 6 is still supported by NativeDoc, but on IE 6, the documentation will not have the JavaScript tree, instead a classic list is displayed.

The main reason of removing the JSTree on IE 6 is that NativeDoc 1.5.1 templates were meeting sometimes problems with latest versions of Opera, in order to fixe those problems the jsTree library has been updated to version 0.9.9a, all problems has gone with Opera and all modern browser, but on Internet Explorer 6, NativeDoc website were not rendered correctly.

> If you really want to have the JavaScript tree on IE 6, use NativeDoc 1.5.1.
