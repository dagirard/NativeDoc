NativeDoc Script Comments Cleaner


NativeDoc has a "Script Comments Cleaner". Its goal is to reduce the size of your stacks by removing all comments from them.

Usage:


The main function of the "script comments cleaner" is dpnd_comment_remove(p_Script). This function simply returns the script p_Script without any comments, and indentation. (Useful to compare two versions of the same source code).

The main DANGEROUS command of the "script comment cleaner" is "dpnd_CompileStack p_MainStack, p_NoConfirmationDialog".

This command simply "compiles/removes all comments" from the script of each objects/cards/substacks of stack mainstack p_MainStack. This is why it is dangerous, because a bad call to this command can destroy all your comments!

This is why there is a confirmation dialog by default, to prevent any bad calls to that command.


Implementation


If you are using "SavingStandalone" to build your application, simply place dpnd_CompileStack inside it.

Example:

on savingStandalone
  dpnd_CompileStack "myMainstack"
end savingStandalone

If you are using your own "builder", simply call dpnd_CompileStack with the name of your mainstack to "compile" source scripts.

WARNING: dpnd_CompileStack DESTROYS ALL YOUR COMMENTS! YOU MUST KNOW WHAT YOU ARE DOING BEFORE USING IT.


Removing the confirmation dialog


If you are really sure of what you are doing, you can remove the confirmation dialog, by setting the parameter <p_NoConfirmationDialog> to "iknowwhatidoletmedothat"

Example:

on savingStandalone
  -- No confirmation dialog.
  dpnd_CompileStack "myMainstack", "iknowwhatidoletmedothat"
end savingStandalone
